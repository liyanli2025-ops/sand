# Tabriz 场景视频资源

本文件夹用于存放大不里士巴扎作坊（猫拆地毯）4 段视频。

## 文件命名规范

| 文件名 | 用途 | 时长建议 | 播放方式 |
|---|---|---|---|
| `bazaar_v1.mp4` | V1 推进：特写梭子织地毯 → 拉远到全景 | 5~8s | 一次性 |
| `bazaar_v2.mp4` | V2 待机：猫爪对毛线球跃跃欲试 | 5~6s | **首尾帧一致，循环** |
| `bazaar_v3.mp4` | V3 触发：猫踢飞毛线球，毛线被拉扯 | 5~6s | 一次性 |
| `bazaar_v4.mp4` | V4 崩坏：地毯散开 → 织机倾倒 → 渐黑 | 6~8s | 一次性，末帧渐黑 |

## 视频规格

- **比例**：9:16 竖屏（1080×1920）
- **编码**：H.264 (libx264)
- **帧率**：24fps 或 30fps，4 段必须一致
- **音频**：无（前端 muted autoplay）
- **构图安全区**：所有关键元素（猫、毛线球、织机）务必落在画面**中央竖向 60% 区域**，左右各预留 20% 缓冲（横屏下会被裁掉）

## 衔接约束（关键！）

为了让 4 段视频"看起来像一段连续画面"，必须严格做到：

1. **V1 末帧 = V2 首帧 = V2 末帧**：全景定镜，织机在上、猫在右下、毛线球在下方中央
2. **V2 首帧 = V2 末帧**：必须完全一致，否则 loop 会跳一下
3. **V2 末帧 ≈ V3 首帧**：猫位置不变，毛线球位置不变
4. **V3 末帧 = V4 首帧**：毛线球已被踢飞，毛线绷紧从地毯里抽出
5. **V4 末帧**：全黑（亮度 0）

推荐做法：先用 AI 生图工具生成 4 张关键帧图（图 1~图 4），再用"图生视频"模式分别做 4 段，
利用工具的"首帧约束"和"尾帧约束"功能保证衔接。

## 部署说明

⚠️ **mp4 文件不会被打包到 H5（page-deploy 脚本会跳过 mp4）**，必须单独上传到腾讯 CDN：

1. 把 4 段视频上传到 `mat1.gtimg.com/qqcdn/redian/sand_test/` 路径下
2. 在 `app.js` 顶部找到 `TABRIZ_VIDEO_URLS` 常量，把 4 个 URL 改为正式 CDN 地址
3. 本地调试时可临时改成 `./videos/bazaar_v1.mp4` 用相对路径
